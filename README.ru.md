# File DB Manager

[![GitHub release](https://img.shields.io/badge/релиз-v1.1.0-blue.svg)](https://github.com/Mishin-18/file-db-manager/releases/tag/v1.1.0)

🇬🇧 [English version](README.md)

Профессиональный инструмент для индексации и управления файлами на базе SQLite.  
Сканируйте папки, отслеживайте файлы, сверяйте списки путей с базой и вручную разрешайте неоднозначности.

---

## ✨ Возможности

- 📁 **Индексация файлов в SQLite** (имя, путь, размер, хэш, дата изменения)
- 📂 **Сканирование одной папки или нескольких невложенных папок** в единую базу
- 🔄 **Инкрементальное обновление базы** без полного пересоздания
- 🔍 **Нечёткий поиск** с Unicode-нормализацией
- 📊 **Сравнение списков путей** из буфера обмена, TXT и Excel
- 🎯 **Три статуса**: `FOUND` / `MISSING` / `AMBIGUOUS`
- 🌐 **Интерфейс с 6 языками** и переключением на лету
- 🇨🇭 **Поддержка немецкого, французского, итальянского и ретороманского языков**, актуальных для Швейцарии
- 📈 **Расширенное окно прогресса** с общим числом файлов, отображением обработано/всего, процентом и прогресс-баром
- 🖥️ **Интеграция с Windows** — "Показать в папке" с выделением файла (Win10/Win11)
- 🧪 **Автоматизированные тесты на pytest** — текущий локальный статус: **12 passed**

## 🧱 Эволюция проекта

В этом репозитории показаны три этапа развития проекта:

1. **Исходный монолит**  
   Первая рабочая однофайловая версия приложения.

2. **Подготовленный монолит**  
   Переходная однофайловая версия, перестроенная для более безопасного будущего разделения на модули.

3. **Многомодульная версия**  
   Рефакторинг в архитектуру из нескольких модулей как следующий этап развития проекта.

### Почему это важно

Целью было не только добавление функций, но и поэтапное улучшение внутренней структуры проекта.

Эта эволюция показывает:
- сохранение исходной рабочей реализации
- промежуточный этап рефакторинга вместо рискованной полной переписки
- переход к многомодульной архитектуре
- более аккуратную организацию тестов
- внимание к поддерживаемости и понятности проекта

## 📁 Структура проекта

- `file_db_manager.py` — исходная монолитная версия
- `prepared_monolith_for_split.py` — переходная монолитная версия, подготовленная к будущему разделению на модули
- `modular_version/` — многомодульная версия приложения
- `tests/` — набор автоматизированных тестов
- `screenshots/` — скриншоты приложения

## 🚀 Быстрый старт

### Требования
- Python 3.10 или выше

### Установка

```bash
git clone https://github.com/Mishin-18/file-db-manager.git
cd file-db-manager
pip install -r requirements.txt
```

### Запуск исходного монолита

```bash
python file_db_manager.py
```

### Запуск многомодульной версии

```bash
python modular_version/main.py
```

### Запуск тестов

```bash
python -m pytest -q
```
